## 1. 게임 개요

| 항목 | 내용 |
|---|---|
| 장르 | 방치형 육성 시뮬레이션 |
| 테마 | 스트리머(이베코짱)의 방송국을 매니저로서 운영 |
| 핵심 루프 | 방송 ON → 재화 획득 → 가챠(영업) → 아이템으로 케어 → 더 효율적인 방송 |
| 성장 축 | ① 구독자(채널 체급) ② 매니저 레벨(유대감) — 이원화 |

## 2. 재화 및 스테이터스

### 2-1. 구독자 (Subscribers)

| 항목 | 설명 |
|---|---|
| 역할 | 타 게임의 "전투력"에 해당. 매니저 레벨 승급의 필수 조건 |
| 성격 | **절대적 성장 지표 — 감소 없이 누적만 됨** |
| 증가 조건 | 방송 ON 상태일 때 시간에 따라 지속 증가 (미접속 시에도 증가) |
| 감소 조건 | 없음 |
| 표기 규칙 | [6-4. 숫자 표기 규칙](#6-4-숫자-표기-규칙-formatting-rules) 참고 |

### 2-2. 방송 자금 (Broadcast Funds)

| 항목 | 설명 |
|---|---|
| 역할 | 소비형 재화. 매니저의 활동비(교통비, 식비, 미팅비 등) 개념 |
| 증가 조건 | 방송 ON 상태일 때 시간에 따라 지속 증가 (미접속 시에도 증가) |
| 감소 조건 | 가챠(영업) 이용 시 소모 · 텐션 0일 때는 수익이 거의 0에 수렴 |

### 2-3. 텐션 (Tension) — 최대 100

| 항목 | 설명 |
|---|---|
| 역할 | 스트리머의 기분·감정 곡선. 방송 수익 효율을 직접 결정 |
| 0이 되면 | "잠방/눕방" 상태. 구독자 소폭 상승, 자금 수익 거의 0 |

**증가 수단:**

| 수단 | 회복량 | 제한 |
|---|---|---|
| 옷 갈아입히기 | 텐션 Max(100)까지 즉시 회복 | 하루 1회 |
| 디저트 사용 | 아이템별 상이 | 가챠 획득 의존 |
| 소통(터치) | 하루 총 200 회복 가능 목표 <!-- 추가 --> | 에너지 충전식, 일일 상한 |
| 방송 OFF 휴식 | 1분당 +1 (약 1.6시간에 0→100) | — |

**감소 요인:**

| 요인 | 상세 |
|---|---|
| 방송 피로 | 방송 ON 시 시간 경과로 하락 |
| 과도한 터치 | 에너지가 없는 상태에서 반복 터치 시 하락 |

## 3. 시스템 설계

### 3-1. 방송 토글 (▶ LIVE)

| 상태 | 효과 |
|---|---|
| **ON** | 구독자/방송 자금이 시간에 따라 증가. 텐션 소모 (3분당 -1)|
| **OFF** | 성장 멈춤. 텐션이 시간에 따라 회복 (1분당 +1) |

### 3-2. 영업 · 가챠 (Sales & Gacha)

방송 자금을 투입하여 매니저가 외부 협찬을 따오는 **'활동비 지출' 개념의 가챠**.

| 항목 | 내용 |
|---|---|
| 비용 | 1회 **100 재화** / 10연 **1,000 재화** |
| 무료 혜택 | 매일 **무료 10연 1회** (00시 초기화) |
| 광고 보상 | 광고 시청 1회당 **무료 10연 추가** · 일일 **2회** 기회 |
| 일일 무료 합계 | **최대 30회** (기본 10 + 광고 20) |
| 보상 | 의상(영구 자산) 또는 소비 아이템(음식, 디저트, 버프템) |
| 연출 | SSR 확정 시 매니저 전용 자신감 대사 출력 |

> **천장(Pity) 시스템
> - 가챠 누적 **100회** 시 의상 아이템 하나 확정
> - 일일 무료 가챠로는 누적되지 않음
> - 자금 가챠 1회 1회가 "투자"처럼 느껴지는 구조

**중복 의상 환전:**

| 등급 | 환전 재화 |
|---|---|
| Common | 10 재화 |
| Rare | 20 재화 |
| Legendary | 100 재화 (가챠 1회분 리턴) |

### 3-3. 케어 (Care)
- 가챠로 획득한 **소비성 아이템(식사, 디저트)**을 사용하는 전용 메뉴
- **초과분 폐기:** 텐션 100 상태에서도 사용 가능하나 100 초과분은 버려짐

### 3-4. 세팅 · 옷장 (Setting / Wardrobe)
가챠로 획득한 의상을 관리·교체하는 메뉴. 획득 시 바로 옷장에 추가됨.

| 슬롯 | 효과 |
|---|---|
| **헤드(Head)** | 방송 자금 획득량(%) 보너스 |
| **바디(Body)** | 구독자 증가량(%) 보너스 |
| **세트 효과** | 동일 세트 장착 시 텐션 하락 속도 감소 |

- 하루 **1회** 의상 교체 시 → **텐션 Max(100) 즉시 회복** + 매니저 경험치 획득

### 3-5. 소통 · 터치 (Touch)

<!-- 추가: 에너지 충전 방식 -->

| 항목 | 내용 |
|---|---|
| 방식 | 에너지 충전식 |
| 충전 속도 | 1시간에 n회 충전 (미확정) |
| 최대 보유량 | 제한 있음 (예: 5회 충전 / 20회 보유 — 미확정) |
| 일일 텐션 회복 목표 | 합계 **200** <!-- 추가 --> |

| 과도한 터치 | 에너지 없이 반복 시 텐션 하락 + 싫어하는 반응 연출 |

### 3-6. 매니저 레벨 & 호감도 (Manager Level)

<!-- 수정: 성장 이원화 구조 명확화 -->

**성장 이원화:**

| 축 | 역할 | 올리는 방법 |
|---|---|---|
| **구독자 수** (채널 체급) | 레벨업의 **승급 조건** | 방송 ON 시 자동 누적 (감소 없음) |
| **매니저 XP** (유대감) | 레벨업의 **경험치** | 옷 갈아입히기, 식사 제공, 소통(터치) |

- 레벨 상승 시: 방송 효율(구독자/자금 획득량) 영구 버프 획득

### 3-7. 개인 통합 메뉴 (Manager's Dashboard)

메인 화면의 **매니저 레벨(하트 아이콘)** 클릭 시 노출되는 통합 관리 모달. 유저의 성장 지표와 데일리 과제, 그리고 데이터 분석을 3개의 탭으로 제공한다.

#### **[Tab 1] 커리어 & 승급 (Career)**

매니저로서의 정체성과 장기적인 목표를 확인하는 탭이다.

| 항목 | 상세 내용 | 비고 |
| :--- | :--- | :--- |
| **매니저 등급/레벨** | 현재 직함 및 레벨 표시 (예: 베테랑 매니저 Lv.15) | 레벨에 따른 직함 변화 |
| **승급 조건** | 다음 등급까지 필요한 구독자 수 ($Target - Current$) | 진행도 바(Progress Bar) 병행 |
| **등급 전용 버프** | 매니저 숙련도로 인한 '영구 고정 버프' 수치 명시 | 예: 구독자 $+15\%$, 자금 $+10\%$ |

#### **[Tab 2] 데일리 케어 (Daily)**

오늘의 숙제와 이베코짱의 실시간 상태를 체크하는 탭이다.

| 항목 | 상세 내용 | 비고 |
| :--- | :--- | :--- |
| **일일 업무 현황** | 식사 제공($n/4$), 의상 세팅($n/1$), 터치 에너지 잔량 | 초기화 시간(00:00) 표시 |
| **상태 요약** | 포만감(Stamina) 및 텐션(Tension)의 구체적 수치 | 메인 UI보다 정밀한 숫자 표기 |
| **오늘의 코멘트** | 현재 상태에 따른 이베코짱의 짧은 한마디 | "오늘 컨디션 최고예요\!" 등 |

#### **[Tab 3] 채널 분석 (Analysis)**

모든 버프가 합산된 '최종 효율'을 뜯어보는 전략 탭이다.

| 항목 | 상세 내용 | 비고 |
| :--- | :--- | :--- |
| **전체 버프 합산** | **Total Buff List**<br> - 매니저 레벨 보너스: $+15\%$<br> - 장착 의상 보너스: $+5\%$<br> - 소비 아이템 효과: $+20\%$ (남은 시간 표시) | 요인별 세부 리스트업 |
| **실시간 성장 지표** | **성장률 데이터**<br> - 구독자 증가율: $+1,250$ Subs / min<br> - 예상 후원금: $+54,000$ DONO / min | 현재 텐션 구간 반영 결과 |

### 4-3. 방송 수익 엔진 (LIVE ON)

<!-- 수정: 피버 배율 정상화 -->

| 텐션 구간 | 상태 명칭 | 3분당 자금 | 3분당 구독자 | 비고 |
|---|---|---|---|---|
| **90 ~ 100** | 피버 모드 | **30** | **500** | 알고리즘을 탄 최고 상태 |
| **70 ~ 90** | 하이 텐션 | **10** | **150** | 최적의 관리 상태 |

> **구간 간 배율 (수정 후):**
> - 피버 vs 하이: 자금 **3배**, 구독자 **3.3배** ✅
> - 피버 vs 보통: 자금 **6배**, 구독자 **5배** ✅

SUBS_HIGH_PER_SEC:         3 / 180,      // 텐션 70~90
SUBS_MID_PER_SEC:          2 / 180,      // 텐션 30~70

var TEST_TIME_SCALE           = 1;          // 테스트용 배속 (10이면 10배속)

## 6. UI / 연출 사양

### 6-1. 현실 시간 동기화 (오프라인 보상)

- 미접속 시에도 방송 상태에 따라 정산 처리
- 낮/밤 배경 전환 및 OFF 시 낮잠 연출 (안대 착용 등)

### 6-2. 매니저 종합 인포 대시보드 (3-Tab)

하트 모양의 매니저 레벨 아이콘 클릭 시 노출.

| Tab | 이름 | 내용 |
|---|---|---|
| 1 | **커리어** | 현재 등급, 레벨(XP), 다음 승급까지 필요 구독자 목표, 등급 전용 영구 버프 수치 |
| 2 | **데일리** | 오늘의 케어 현황(식사 0/4, 세팅 0/1 등), 터치 에너지 잔량 �� 충전 시간 |
| 3 | **분석** | 현재 적용 중인 전체 버프 합산 리스트, 실시간 시간당 구독자/방송 자금 획득 지표 |

### 6-3. SSR 연출

- 등급이 높은 협찬(Legendary) 확정 시 매니저의 자신감 넘치는 **전용 대사** 출력

### 6-4. 숫자 표기 규칙 (Formatting Rules)

전광판(`.live-value`, `.earnings-value`) 출력 규칙.

| 조건 | 표기 방식 | 예시 |
|---|---|---|
| 7자리 이하 (0 ~ 9,999,999) | 3자리마다 콤마 | `1,234,567` |
| 8자리 이상 (10,000,000~) | 단위 축약 (M, B, T) | `10.5M` |

- $10^6$: **M** · $10^9$: **B** · $10^{12}$: **T**
- 소수점 첫째 자리까지 표시하되, `.0`이면 정수만 노출
